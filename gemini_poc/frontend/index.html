About AlienVox

AlienVox

v0.1.0 — Proof of Concept
Lightweight tray-first "Speak Selection" utility for Windows & macOS.

What is AlienVox?

AlienVox fills a gap that has existed for decades: Windows still lacks a simple, universal "Speak Selection" feature like macOS's Option + Esc. Highlight any text and hear it spoken instantly — no extra paragraphs, no UI wrappers, no heavy screen readers.

This prototype is the first step toward that experience: a lightweight, tray-first desktop app with local TTS engines and a clean path to neural voices.

Tech Stack

  • Language: Rust (core native app)
  • Framework: Tauri 2.x — WebView2 on Windows, WRY cross-platform runtime
  • System Tray: Win32 Shell_NotifyIconW (Windows), TAO system-tray (macOS)
  • Frontend: Vanilla HTML / CSS / JS — no framework, classic Win32/WPF aesthetic
  • Build: Cargo + Tauri CLI; Python only for dev tooling (setup.py, run.py)

TTS Engines & Models

  • Kokoro-82M — Primary local neural TTS target. Open weights, Apache 2.0, ~82 M parameters.
  • Native OS TTS — Windows SAPI 5 / WinRT SpeechSynthesis; macOS AVSpeechSynthesizer. Always available as the reliability floor.
  • Piper — Small offline neural TTS (MIT). Archived upstream; treated as a stable fallback.
  • VibeVoice-Realtime-0.5B — Streaming local TTS experiment (MIT, research stage).
  • Zonos / ZONOS2 — High-fidelity local experiment with voice cloning (Apache 2.0, GPU-oriented).
  • Dia — Expressive dialogue generation (Apache 2.0, GPU-oriented).

Cloud APIs (Optional)

Cloud engines are not required for core functionality but may be used for experiments:

  • Gemini TTS — Google's multimodal TTS API.
  • ElevenLabs — High-quality neural voices (paid / rate-limited).
  • OpenAI TTS — GPT-SoVITS-compatible API (paid).

Architecture

AlienVox follows a bridge pattern with strict platform isolation:

  • src/main.rs — Win32 tray icon, context menu, Tauri AppHandle bridge.
  • src/audio/ — Platform-specific audio playback (Win32 / macOS).
  • src/capture/ — Text selection capture via UI Automation (Windows) or AXUIElement (macOS).
  • src/engines/ — TTS engine abstraction layer (ML models, native OS engines).
  • src/platform/ — Platform-specific main entry points.
  • frontend/ — WebView2 UI panel with classic Win32 aesthetic (toolbar, sliders, voice controls).

Design Philosophy

  • Frictionless execution: Zero config for the primary path — global hotkey → speak selection.
  • Strict respect for selection: Only the highlighted text is spoken, nothing more.
  • Hybrid audio pipeline: Local zero-latency APIs first; clean path to cloud/AI voices when online.
  • Unobtrusive presence: Lives in the system tray — no active window during operation.
  • Privacy-first: Captured text is never logged, cached, or persisted locally.

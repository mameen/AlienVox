## ADR-001: Selection of Tech Stack for AlienVox (gemini_poc)

## Context and Problem Statement
The **AlienVox** utility requires ultra-low latency execution (<150ms from global keystroke to audio initialization) and a highly compressed system resource footprint (target sub-10MB idle RAM usage). The application must run uniformly across Windows 11 and macOS 14+ while remaining running silently in the background (System Tray / Status Menu Bar). 

## Decision
We evaluated several technical avenues, specifically weighing modern web-desktop runtimes (Node.js/Electron), native platform segmentation (C#/.NET Core + Swift), and a unified low-level systems programming compilation model (**Rust + Tauri**). Additionally, we needed to ensure full capability to run or bind state-of-the-art (SOTA) Text-to-Speech (TTS) models directly on client hardware without introducing runtime interpreter bottlenecks.

**Selected: Rust + Tauri** for the gemini_poc prototype.

## Why This Record Is Here
This ADR is specific to the `gemini_poc` Rust + Tauri POC. It does not apply to other implementations under `tts/`. The outer project docs remain technology-agnostic; implementation decisions live here.

## Superseded By
The Python + PySide6 implementation in `tts/python_app/` supersedes this POC. See `python_app/docs/adr/adr-001-python-pyside6-stack.md`.
